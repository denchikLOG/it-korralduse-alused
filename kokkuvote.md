# Kokkuvõte

Git on versioonihaldussüsteem, mida kasutatakse failide ja projektide muudatuste jälgimiseks. Git võimaldab töötada meeskonnas ning hoida kogu projekti ajalugu ühes kohas. GitHub on veebiplatvorm, kus saab Git repositooriume hallata ja teistega jagada.

Selle töö käigus õppisin looma GitHubi profiili ja repositooriumi. Samuti õppisin kasutama README faile dokumentatsiooni koostamiseks. Mõistsin, et hea dokumentatsioon aitab projekti paremini mõista ja hallata.

Commit on muudatuse salvestamine Git repositooriumi ajalukku. Push tähendab kohalike muudatuste saatmist GitHubi serverisse. Seega commit salvestab muudatused lokaalselt ja push avaldab need kaugserveris.

Harud ehk branches on kasulikud, sest need võimaldavad teha muudatusi ilma põhiversiooni mõjutamata. Arendajad saavad töötada erinevate ülesannete kallal samaaegselt. See muudab meeskonnatöö lihtsamaks ja turvalisemaks.

Pull Request on tööriist, mille abil saab muudatusi ühest harust teise liita. Enne ühendamist saab muudatusi üle vaadata ja kontrollida. See aitab vältida vigade sattumist põhiversiooni.

Git erineb tavalisest failide kopeerimisest selle poolest, et kogu muudatuste ajalugu säilib. Vajadusel saab varasema versiooni taastada. Lisaks on lihtne näha, kes ja millal muudatusi tegi.

GitHub toetab ka projektijuhtimist. Issues võimaldavad hallata ülesandeid ja probleeme. Pull Requestid aitavad muudatusi kontrollida. Need tööriistad muudavad projektide juhtimise organiseeritumaks.

Mulle meeldis selle töö juures kõige rohkem GitHubi kasutamise õppimine. Kõige keerulisem oli harude loomine ja Pull Requestide ühendamine. Pärast harjutamist muutusid need tegevused siiski arusaadavaks.

Tulevikus kasutaksin GitHubi nii õpingutes kui ka tööelus. See on oluline tööriist tarkvaraarenduses, süsteemihalduses ja projektijuhtimises. Usun, et Git ja GitHub on oskused, mida vajab iga IT-spetsialist.
